RPi Konfiguration

Als DICOM Server wird in diesem Tutorial die Software Orthanc dienen. Diese wird auf dem Raspberry Pi installiert, sowie auch auf dem PC. Orthanc kann als Server und als Modalität1 konfiguriert werden. Auf dem RPi wird sie als Server genutzt und auf dem PC als simulierte Modalität.

Installation

RPi:

  1. Nach Verbindung via SSH2 wird folgender Befehl eingegeben.

    sudo apt-get install orthanc

  2. Nach erfolgreicher Installation wechseln wir das Verzeichnis, um zur Konfigurations Datei des DICOM Servers zu gelangen.

    cd /etc/orthanc

  3. Anschliessend öffnen wir diese Datei um die Konfiguration vorzunehmen mit folgendem Befehl.

    sudo nano orthanc.json

    Zur Navigation werden die Pfeile auf der Tastatur eingesetzt.

Die Konfigurationsdatei auf dem PC ist eine Text-file und identisch mit dem Beispiel auf dem RPi. Dabei gilt zu beachten, dass diese als Administrator ausgeführt werden muss zur erfolgreichen Speicherung der Änderungen.

Inbetriebnahme

Konfiguration des Http Servers

Die Nummer des “HttPort” kann frei gewählt werden und ist für dieses Tutorial weniger relevant. In unserem Fall vergeben wir die Nummer: 8044

Konfiguration des DICOM Servers

In der Konfiguration des DICOM Servers sind die Parameter “DicomAet” und “DicomPort” von Bedeutung. Sie werden benötigt um eine erfolgreiche Anbindung im DICOM Netzwerk zu ermöglichen.

Anbindung einer Modalität

Das ist der wichtigste Teil. Nicht von der Konfiguration des Servers, sondern vielmehr für eine erfgolreiche Anbindung einer DICOM AE allgemein.

Diese Parameter werden benötigt um jedes bildgebende System oder bildverarbeitende Software als “Knoten” im DICOM Netzwerk anzubinden.

Wenn alle Änderungen vorgenommen wurden, kann mit folgender Tastenkombination die Datei abgespeichert und verlassen werden.

  • Ctrl + S “speichern”
  • Ctrl + X “verlassen”
